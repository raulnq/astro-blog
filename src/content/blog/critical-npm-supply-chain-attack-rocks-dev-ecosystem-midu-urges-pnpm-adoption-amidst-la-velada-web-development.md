The software development community is on high alert following a significant supply chain attack targeting the npm ecosystem, with developer Midu highlighting the severe compromise of key libraries like Tstack. The attack involved an adversary publishing 84 malicious versions across 42 different packages, including popular ones like TanStack Query (with billions of downloads) and TanStack Start. The malware's payload included exfiltrating sensitive environment variables (AWS, Google Cloud, GitHub, SSH), uploading files, and propagating itself. Alarmingly, the attacker implemented a 'kill switch': if exposed tokens were revoked, the malware would initiate an `rm -rf` command, wiping the infected machine's hard drive. This exploit leveraged a vulnerability in GitHub Actions' `pull_request_target` event, allowing cached contamination without direct npm token compromise. Midu strongly advocated for adopting PNPM v11, which offers default supply chain protection features like a minimum release age of 1440 minutes (one day), blocking exotic sub-dependencies, and preventing pre/post-install scripts from executing, thereby mitigating many common attack vectors. Similar attacks were noted across other ecosystems, including RubyGems and Python, underscoring a broader industry challenge.

Amidst these security concerns, Midu commenced the live development of the official 'La Velada del Año 6' website, an open-source project sponsored by Infojobs. The development process heavily utilized AI coding assistants such as Cursor, Codex, and Opus 4.7 for tasks ranging from component generation to CSS styling, even engaging in humorous 'negotiations' with the AI over design choices. A significant job opportunity tied to this project garnered over 30,000 applicants, with Infojobs hiring two and Midu personally sponsoring two additional positions. Midu also teased upcoming projects, including a website for a 'World Cup-winning international footballer.' The live stream experienced intermittent technical difficulties, including screen freezes and connection issues, which Midu attributed to network problems or heavy agent usage, sparking real-time community interaction and troubleshooting. Looking ahead, Midu announced an upcoming talk in Murcia and a live stream of the Google I/O keynote next week, continuing to foster engagement within the developer community.
